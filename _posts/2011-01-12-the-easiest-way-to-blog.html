What's Wrong With Wordpress?

Nothing.
But the Jekyll mindset / philosophy is more like the future.

Wordpress is an opinionated Content Management System — like Rails is for web development &mdash focusing on extendability.
It has a well thought, built-in rendering logic to display different content, and, a hook / callback system to enable plugins.
And a huge community creating infinite number of themes and plugins.

Creating simple websites with Wordpress is very productive, running them is very stable.

In 5 years of hosting and running many Wordpress sites — from simple blogs to complete e-commerce systems — I've never had a system failure or a bug affecting the site uptime.
Setting up a site takes a few minutes, all upgrades are just a button away and invisible to the users.
Adding widgets or extending WP — seo, maps, social sharing and streams, navigation,etc. — is almost immediate, there are plugins for everything, the hard thing is to choose the best.

Customers / users with average net literacy are happy with the Admin interface, the documentation is exhaustive and translated into all languages, the learning curve is fast.

Wordpress as a brand has a good acceptance from customers by being perceived as the most popular blogging engine today.

But You Can Do Better ...

Yes, I have to admit that I've became locked in by my vendor Wordpress.

The Wordpress developer's way is nothing more than:

  • Altering the standard rendering logic (controllers) if the new site requires.
  • Using the same plugins all over again.
  • Tweaking themes using only plain CSS/HTML.
  • Hacking PHP code to extend the Wordpress API, or, to make plugins work seamlessly together.
  • Working around the natural limits the framework is forcing on you.

... so developing Wordpress sites was no more fun after a while.

I'm rather developing something than gluing together parts all over again.
And I would sleep better knowing I have full control over my code and I can take any challenge comming from my customers.

Enter Jekyll

It was a random try to run this blog on Jekyll and I was immediately fallen in love ... here is the why.

Fast, Stable, Secure and Cheap

There is no faster, cheaper and more secure solution to run a site than the Jekyll way — by serving static files directly from the file system.

One can opt for caching to reach the same performance but the caching middleware needs attention, thus resources both from the developer and the server.

Fun

Yes, (beeing Rubyists) we are looking for fun all the time. Writing articles in your code editor and deploy them via command line is our second nature.

Quickly adding new features to Jekyll via it's fantastic plugin system makes us feel good — with every article written we are creating our own blogging system too.

Open

No more vendor locking. In fact, I'm glad to be locked in Ruby.
And confident my blog engine can be extended easily by any means.

More, I can now create my own CMS based on the pros and cons I've learnt using Wordpress, the de facto standard in blogging.

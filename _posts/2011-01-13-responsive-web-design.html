The Short History of Responsive Web Design

{{ page.people[0].name }} in {{ page.links[2].title }} article, back in May 2010 opened my eyes about a new, and in the same time old technology capable to produce web sites compatible with various displays like desktop monitors, tablets, smartphones and smaller mobile devices.

After reading the article I was surprised how straightforward and easy is to create responsive web sites.

You just have to use elastic measures (em, %) in your CSS and add two lines of code in your HTML. There is a simple magic formula how to size design elements relatively to the current display resolution and that's all, the rest is done by the modern browser.

A few months later {{ page.people[1].name }} the grandpa of all web designers stated {{ page.links[1].title }} so I was sure I'm on the right track.

With the {{ page.links[0].title }} Responsive Web Design got full respect and support from the designer elite.

So please forget responsiveness is not the silver bullet — just flow with the opinion leaders.

And do not trust ideas like "Mobile Web Design in the Rails Way" &mdash web development frameworks are opinionated about development not about design.

Yes, The Images

CSS media queries and the flexible grid &mdash the foundation for responsiveness &mdash are dealing with the display size of the HTML elements in a webpage and not with the size of images in kilobytes to be downloaded.

It sounds good to use differently sized (in KB) images for desktop and mobile. You save bandwith and rendering time.

Perhaps everybody uploads different sizes for an image &mdash thumbnail, medium, large &mdash by default.

So it remains to detect run-time &mdash with Javascript and cookies &mdash which image size to use for different displays.
